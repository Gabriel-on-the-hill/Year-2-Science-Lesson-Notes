Welcome

Build a Flying Parrot Game

A parrot, some buildings, a score, and three difficulty levels — built one piece at a time.

What you'll end up with

A parrot that flaps with the U key, falls with gravity, and dodges buildings that scroll across the screen. A score counts every building you survive. Press 1, 2, or 3 for easy, medium, or hard. Press R to play again.

Phase 1

Make the parrot fly

A parrot that flaps up when a key is pressed, and falls down on its own.

★ New idea: Gravity loop

A loop that keeps moving the parrot down a tiny bit, over and over. Pressing a key pushes it up a lot, but only once. The push and the loop fight each other — that's what flying feels like.

Try the gravity demo below to see how it works before building it in Scratch. Click the flap button. Then click it lots. Then stop clicking.

🧪 Demo: how gravity feels
Phase 1 · Build

Build the gravity and flap

  1. From the sprite library, add a Parrot. Check it has two costumes (wings up and wings down). If not, pick a sprite that does.
  2. Click the parrot. Add this script — sets the start position, then runs gravity forever:
    when 🟢 clicked
    go to x: -100 y: 0
    forever
    change y by -3
    wait 0.1 seconds
  3. Add a second script — the flap:
    when u key pressed
    change y by 20
    switch costume to parrot-b
🧪 Try it

Click the green flag. The parrot should drift down. Tap U a few times — it should jump up each time.

✓ Check yourself
  • Does the parrot fall on its own?
  • Does it jump up when U is pressed?
  • If you stop tapping U, does it eventually leave the screen at the bottom? (Yes — that's normal for now.)
Phase 2

Buildings appear and scroll past

Buildings keep spawning on the right and sliding to the left.

★ New idea: Clones

A clone is a copy of a sprite, made while the game is running. One "parent" stays hidden and makes a fresh clone every few seconds. Each clone has its own little life — it appears, moves, then disappears.

🧪 Demo: cloning in action

Each new building is a fresh clone of the same hidden parent.

Phase 2 · Build

Build the buildings

  1. Add a new sprite — any building or block shape. Give it a few costumes if you want variety.
  2. The "parent" building hides itself and spawns a clone every 2 seconds:
    when 🟢 clicked
    hide
    forever
    wait 2 seconds
    create clone of myself
  3. Add the script that runs on every new clone:
    when I start as a clone
    go to x: 240 y: 0
    switch costume to pick random 1 to 4
    show
    repeat until touching edge ▼ ?
    change x by -3
    delete this clone
🧪 Try it

Click the flag. Every 2 seconds a new building should appear on the right and slide across. Experiment: change wait 2 to wait 0.5. What happens?

✓ Check yourself
  • Do buildings appear from the right?
  • Do they slide left across the screen?
  • Do they vanish when they reach the left edge?
Phase 3

Crash! 🎯 Solo Mission

Time to figure something out without step-by-step instructions.

🎯 Your mission

Right now the parrot can fly straight through buildings — that's not a game. Change the parrot's script so the game ends when the parrot crashes into a building. Show a "Game Over!" message too.

🧰 You already know everything you need
  • repeat until ... — runs again and again, then stops when something becomes true
  • touching sprite ▼ ? — true when this sprite touches another sprite
  • say Hello for 2 seconds
  • broadcast message ▼ + stop other scripts in sprite

Think: the parrot's gravity uses forever. What if it wasn't forever — what if it ran until something happened?

Phase 3 · Hints & Solution

Stuck? Open a hint 🎯 Solo

Open them one at a time, only if needed.

Hint 1 — where should the change happen?

Don't change the buildings. Change the parrot's gravity script. Gravity is what makes the parrot keep moving — if it stops, the game stops.

Hint 2 — what replaces forever?

Use repeat until. Inside the diamond shape, drop the question: "am I touching the building sprite?"

Hint 3 — how do I announce game over?

After the repeat until finishes, add say Game Over! for 2 seconds. To stop the buildings spawning too, broadcast Stop first. On the building sprite, react with stop other scripts in sprite.

✓ Check yourself
  • Does the parrot stop falling when it touches a building?
  • Does "Game Over!" appear?
  • Do new buildings stop spawning after the crash? (Bonus.)
One way to do it (open only after you've tried)

Parrot — gravity script:

when 🟢 clicked
go to x: -100 y: 0
repeat until touching Buildings ▼ ?
change y by -3
wait 0.1 seconds
broadcast Stop ▼
say Game Over! for 2 seconds

Buildings — stop on Stop:

when I receive Stop ▼
stop other scripts in sprite ▼

If your version is shaped differently but works the same way — your version is right too.

Phase 4

Keep score

Every time a building goes past, the score goes up by 1.

★ New idea: wait until ...

This block pauses a script and waits — does nothing — until a question becomes true. Like waiting at a door until someone opens it. Once the door opens, the next block runs.

🧪 Demo: wait until count reaches 5

Click the button to count up. The script below is waiting…

0
wait until count ≥ 5
say Hello!
Status: waiting…
Phase 4 · Build

Build the score

  1. In the Variables tab, click Make a Variable. Name it Score. Tick the box next to it — that puts the score on the stage.
  2. On the building sprite, find the green-flag script (the spawner). Add set Score to 0 right after when 🟢 clicked. This resets the score every new game.
  3. Add a second "when I start as a clone" script on the building sprite. Its only job is to wait for a clone to leave the screen, then add to the score:
    when I start as a clone
    wait until x position < -200
    change Score by 1
🧪 Try it

Click the flag. Watch the score in the corner. Every time a building slides off the left, the score should go up by 1.

✓ Check yourself
  • Does Score start at 0?
  • Does Score go up by 1 every time a building exits left?
  • Does Score stop changing after the parrot crashes?
Why this works

Each clone has two scripts running at once. One moves it left. The other just sits and waits — doing nothing — until that clone has passed x = -200 (well off the left). Then it adds 1 to the score. Same sprite, two scripts, no problem.

Phase 5

Pick your level

Press 1, 2, or 3 at the start to choose easy, medium, or hard.

★ New idea: Variables that drive behaviour

So far Score has been a variable that just remembers. Now you'll make two new variables — Speed and Interval — that the game reads from while it plays. Press a different key, the variables change, the game feels different. Same code, different feel.

🧪 Demo: one variable, different behaviours
🦜
Speed: 3

The "fly" code is the same every time. Only the Speed variable changes.

Phase 5 · Build (1 of 2)

Make the variables and player input

  1. Make two new variables: Speed and Interval. Untick their stage boxes — they don't need to show.
  2. On the parrot, replace the green-flag script. Instead of starting the game right away, ask the player to pick a level first:
    when 🟢 clicked
    go to x: -100 y: 0
    say Press 1, 2, or 3 to pick a level!
    wait until <key 1 pressed? or <key 2 pressed? or key 3 pressed?>>
    say (empty — clears the bubble)
    broadcast Start ▼
    repeat until touching Buildings ▼ ?
    change y by -3
    wait 0.1 seconds
    broadcast Stop ▼
    say Game Over! for 2 seconds

Now the parrot waits for a key. When 1, 2, or 3 is pressed, it broadcasts Start. Next slide hooks the buildings up to that broadcast.

Phase 5 · Build (2 of 2)

Hook up the building behaviour

  1. On the building sprite, add three new scripts — one for each level. Each sets Speed and Interval:
    when 1 key pressed
    set Speed to -3
    set Interval to 3
    when 2 key pressed
    set Speed to -4
    set Interval to 2
    when 3 key pressed
    set Speed to -5
    set Interval to 1.5
  2. Change the building's spawn script. Replace when 🟢 clicked with when I receive Start. Replace the fixed wait with wait Interval seconds:
    when I receive Start ▼
    set Score to 0
    forever
    wait Interval seconds
    create clone of myself
  3. In the clone's movement script, change change x by -3 to change x by Speed. Now the buildings move at whatever speed the level set.
🧪 Try it

Click the flag. The parrot will ask for a level. Press 1 — slow, sparse buildings. Press 3 — fast and frequent. The same code runs every time. Only the variables change.

✓ Check yourself
  • Does the game wait until 1, 2, or 3 is pressed?
  • Do the three levels feel meaningfully different?
  • Does Score still go up correctly on every level?
Phase 6

Play again

Press R after a crash to restart without clicking the flag.

★ New idea: Restart, not start fresh

The sprites are already on the stage, the score is showing, leftover building clones are still around. Restarting means resetting what's there, not starting from zero.

  1. On the parrot, add a new script that does what the green flag did, plus a Clear broadcast at the top to wipe leftover clones:
    when r key pressed
    stop other scripts in sprite ▼
    broadcast Clear ▼
    go to x: -100 y: 0
    say Press 1, 2, or 3 to pick a level!
    wait until <key 1 pressed? or <key 2 pressed? or key 3 pressed?>>
    broadcast Start ▼
    repeat until touching Buildings ▼ ?
    change y by -3
    wait 0.1 seconds
    broadcast Stop ▼
    say Game Over! for 2 seconds
  2. On the building sprite, stop the spawner on R and add a Clear handler so every leftover clone deletes itself:
    when r key pressed
    stop other scripts in sprite ▼
    when I receive Clear ▼
    delete this clone
🧪 Try it

Play, crash on purpose, press R. The parrot resets, old buildings vanish, the level prompt comes back.

✓ Check yourself
  • Does R work even while the game is still being played?
  • Are all the old buildings cleared?
  • Does the score reset to 0 on the next round?
  • Can you play five games in a row using only R?

Stretch (Optional)

★ The Squirrel Friend Stretch

A character who sits in the corner and announces the score in their own speech bubble — instead of a plain number on the stage.

★ New idea: Joining words and numbers

The join block glues two pieces of text together. join Score: Score gives "Score: 5" when Score is 5. That's how a character can say something with a number in it.

Why this is a stretch

The game already works perfectly without the squirrel. The plain Score box on the stage shows the score just fine. The squirrel is a design choice — it makes the game friendlier, but it adds a few new pieces to manage. Only add it once Phases 1–6 are working.

Stretch · Build

Build the squirrel Stretch

  1. Add a Squirrel sprite from the library.
  2. Untick the Score variable box on the stage — the squirrel will be the score display now.
  3. Hide the squirrel on green flag so it only appears when the game starts:
    when 🟢 clicked
    hide
  4. When Start is received, set up the squirrel in the corner and say the starting score:
    when I receive Start ▼
    go to x: 180 y: 140
    set size to 60 %
    point in direction -90
    show
    say join Score: Score
  5. Score now changes inside the clone scoring script. After change Score by 1, add a broadcast Update ▼.
  6. On the squirrel, react to Update by re-saying the score with the new number:
    when I receive Update ▼
    say join Score: Score
🧪 Try it

Play. The squirrel should sit in the corner saying "Score: 0", then change its number every time a building goes past.

✓ Check yourself
  • Does the squirrel appear when the game starts?
  • Does it say "Score: 0" at the beginning?
  • Does the number in its speech bubble update when score goes up?
